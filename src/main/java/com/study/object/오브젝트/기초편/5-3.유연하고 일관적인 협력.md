### 메시지와 메서드 분리의 문제점
* 인터페이스에만 의존해야 한다.
* 런타임에는 구체 클래스에 의존해야한다.

### 의존성 주입
* 컴파일 타임 의존성과 런타임 의존성 사이의 차이를 해결하기 위해 외부의 객체가 의존하는 객체를 전달하는 방식을 의존성 주입이라고 부른다.


### 메시지와 메서드 분리의 장점
* 변경의 파급효과를 제어할 수 있기 때문에 기존 코드를 수정하지 않고도 기능 확장 가능
* 일관성 있는 설계
  * 추상화를 이용한 설계
    * 메시지에 기반한 추상화에 의존
  * 고정된 협력 패턴
    * 메시지 기반의 협력 패턴 정의
  * 확장 포인트를 이용한 확장
    * 메시지에 기반한 추상화가 확장 포인트를 정의
  * 확장하고 예측하기 쉬운 설계

### 메시지와 메서드 분리의 이유
* 변경하기 쉬운 코드
